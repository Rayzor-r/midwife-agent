**Edit 1: Add auto_detect_note_type() function** — place it in the "Core generation" section, BEFORE the generate_note() function:

```python
# ── Note type detection ────────────────────────────────────────────────────────

_REFERRAL_SIGNALS = {
    "refer", "referral", "letter to", "obstetrician", "specialist",
    "consultant", "re:", "dear dr", "dear nurse",
}

def auto_detect_note_type(bullets: str) -> str:
    """
    Infer whether the bullet input describes a clinical note or a referral letter.
    Returns "referral" if any referral signal keyword is found (case-insensitive).
    Returns "clinical" otherwise.
    """
    text_lower = bullets.lower()
    for signal in _REFERRAL_SIGNALS:
        if signal in text_lower:
            return "referral"
    return "clinical"
```

**Edit 1: Add _check_output_compliance() function** — place it after auto_detect_note_type(), before generate_note():

```python
# ── Output compliance ─────────────────────────────────────────────────────────

import re as _re

_MARKDOWN_HEADER_RE  = _re.compile(r"^#{1,6}\s")
_BOLD_SECTION_RE     = _re.compile(r"^\*\*[A-Za-z ]+\*\*:?\s*$")
_BULLET_LINE_RE      = _re.compile(r"^[-*]\s")
_SOAP_LABEL_RE       = _re.compile(r"^(Subjective|Objective|Assessment|Plan):\s", _re.IGNORECASE)

def _check_output_compliance(note_text: str) -> dict:
    """
    Check generated note text against ACC/DHB LMC documentation format requirements.
    Returns {"passed": bool, "violations": list[str], "warnings": list[str]}.
    Does NOT suppress the note — compliance result is informational.
    """
    violations: list[str] = []
    warnings:   list[str] = []

    for i, line in enumerate(note_text.splitlines(), start=1):
        stripped = line.strip()
        if not stripped:
            continue
        if _MARKDOWN_HEADER_RE.match(stripped):
            violations.append(f"Line {i}: markdown header '{stripped[:40]}'")
        elif _BOLD_SECTION_RE.match(stripped):
            violations.append(f"Line {i}: bold section label '{stripped[:40]}'")
        elif _BULLET_LINE_RE.match(stripped):
            violations.append(f"Line {i}: bullet point in output '{stripped[:40]}'")
        elif _SOAP_LABEL_RE.match(stripped):
            warnings.append(f"Line {i}: explicit SOAP label '{stripped[:40]}'")

    return {
        "passed":     len(violations) == 0,
        "violations": violations,
        "warnings":   warnings,
    }
```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | Tampering | Prompt injection via bullet input to force referral letter format | accept | The midwife is the sole authenticated user of this single-tenant system. Injection by the midwife herself is not a meaningful threat. Bearer token gate prevents external attackers from reaching the generation endpoint. |
| T-04-03-02 | Information Disclosure | Compliance violation log contains snippet of generated note | mitigate | logger.warning() logs to Railway server logs only (operator access required). Log message truncated to first 40 chars of each violating line — does not log full note text. |
| T-04-03-03 | Denial of Service | Pathological regex input causing catastrophic backtracking | mitigate | All four compiled regexes use anchored patterns (^) with no nested quantifiers — not vulnerable to ReDoS. Input processed line-by-line with no full-document regex. |
| T-04-03-04 | Repudiation | Auto-detection silently changes note type without midwife awareness | mitigate | Resolved note_type is always returned in the response dict. Chat tool returns tool result including note_type. Midwife can see which type was applied and override explicitly. |
