- `curl -s -o /dev/null -w "%{http_code}" -H "Authorization: Bearer <key>" http://localhost:8000/api/documents` returns `200` when `<key>` matches `API_KEY` env var

```python
# ── API key auth middleware ─────────────────────────────────────────────────────

EXEMPT_PATHS = {"/api/health"}

@app.middleware("http")
async def require_api_key(request: Request, call_next):
    path = request.url.path
    if path.startswith("/api/") and path not in EXEMPT_PATHS:
        auth = request.headers.get("Authorization", "")
        if not API_KEY or not auth.startswith("Bearer ") or auth[len("Bearer "):] != API_KEY:
            return Response(
                content='{"detail":"Unauthorized"}',
                status_code=401,
                media_type="application/json",
                headers={"WWW-Authenticate": "Bearer"},
            )
    return await call_next(request)
```

**Decision rationale (per D-01, D-02, D-03):**
- `EXEMPT_PATHS` contains only `/api/health`. `/auth/google` and `/auth/google/callback` do not start with `/api/` so they are naturally exempt. `/` is naturally exempt for the same reason.
- The middleware checks `not API_KEY` first — if the env var is unset, all requests are rejected (fail-closed).
- A Bearer token that is a non-empty string matching `API_KEY` is accepted. No hashing or format validation (per D-04).
- The `WWW-Authenticate: Bearer` response header is standard HTTP practice for 401 responses (per Claude's Discretion).
- Use Starlette middleware (not `Depends()` per route) to protect all `/api/*` routes uniformly without modifying each route handler (per Claude's Discretion).

Do NOT modify any existing route handler. The middleware applies globally.
