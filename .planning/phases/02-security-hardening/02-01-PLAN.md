## Objective

Restrict the CORS policy to the deployment origin and remove the credential-exposing textarea from the OAuth callback page, replacing it with a server-log delivery approach.

## must_haves

### goal
Every live API endpoint requires a valid bearer token, the CORS policy admits only the intended origin, the OAuth callback no longer exposes credentials, and the previously exposed Google credentials have been replaced.

### truths
- `main.py` contains `os.getenv("ALLOWED_ORIGIN", "http://localhost:8000")` in the CORSMiddleware call — no `["*"]` wildcard
- `main.py` `google_auth_callback` contains `logger.info` with `GOOGLE_TOKEN` — no `<textarea>` in the response
- A GET to `/auth/google/callback?code=x&state=y` renders HTML with "Google connected" and instructions pointing to Railway logs, with no JSON token in the page body
