  <action>
Replace the `now()` function body (lines ~753-755 in static/index.html) with the following:

```javascript
function now() {
  const d = new Date();
  const day  = d.toLocaleString('en-NZ', { day: '2-digit' });
  const mon  = d.toLocaleString('en-NZ', { month: 'short' });
  const time = d.toLocaleString('en-NZ', { hour: '2-digit', minute: '2-digit', hour12: false });
  return `${day} ${mon}, ${time}`;
}
```

This produces output like "01 May, 13:30" in the browser's local timezone (en-NZ locale, 24-hour clock, no seconds).

Replace ONLY the function body — the function declaration line `function now() {` and closing `}` stay in place. Do NOT modify `appendUserMsg`, `appendAiMsg`, the `.msg-time` CSS rule, or any other part of the file.

Implementation note: Using three separate `toLocaleString()` calls (rather than a single combined call) avoids locale-specific separator placement differences (some locales insert commas or other separators between day and month unexpectedly). This approach produces a deterministic "DD Mon, HH:MM" string regardless of the user's system locale.
  </action>
